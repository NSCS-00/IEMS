## 1. 项目概述

### 1.1 模组信息

| 项目 | 内容 |
|------|------|
| 模组名称 | 综合能源管理系统 |
| 英文名称 | Integrated Energy Management System |
| 缩写 | IEMS |
| Mod ID | `IEMS` |
| Group ID | `com.dlzstudio.iems` |
| 开发团队 | 等离子工作室 (DLZstudio) |
| 适用版本 | Minecraft 1.21.1 NeoForge |

### 1.2 核心功能

1. **高精度能源单位** - SE (标准能量) 和 GE (通用能量)
2. **能源管理方块** - 存储器、转换器、中继器、广播塔
3. **Web 监控界面** - 浏览器实时查看电网状态，设备管理
4. **核心唯一性系统** - 确保世界中只有一个能源核心

### 1.3 设计原则

1. **核心不由本模组提供** - 核心由其他 MOD 通过 API 注册
2. **中继器/广播塔无能量缓存** - 仅标记连接点
3. **存储器不直接供电** - 核心富余时充能，缺电时反向供电
4. **Web 界面替代 GUI** - 无需额外贴图即可访问

## 3. 核心系统设计

### 3.1 核心唯一性系统

**基本原则**: 世界中只能有一个核心！

#### 3.1.1 多 MOD 冲突处理规则

```
优先级规则:
┌─────────────────────────────────────┐
│ 1. 等离子工作室提供的 MOD 优先       │
│    - 仅有一个：直接使用该核心       │
│    - 有多个：选择第一个注册的       │
│                                     │
│ 2. 没有等离子工作室 MOD             │
│    - 按加载顺序选择第一个           │
└─────────────────────────────────────┘
```

#### 3.1.2 错误处理机制

| 情况 | 处理方式 | 日志等级 |
|------|---------|---------|
| 尝试放置第二个核心 | 阻止操作 | `[IEMS 错误]` |
| 多个 MOD 同时注册 | 选择一个，其他忽略 | `[IEMS 警告]` |
| 世界中出现多个核心 | 游戏崩溃 | `[IEMS 致命错误]` |

#### 3.1.3 核心命名

- **默认名称**: `IEMS-CC` (Integrated Energy Management System - Central Core)
- **中文名称**: 综合能源管理系统 - 中枢核心
- **自定义命名**: 允许其他 MOD 通过 API 设置核心名称

#### 3.1.4 日志格式

所有日志使用统一格式：`[IEMS 错误等级] 消息内容`

| 等级 | 前缀 | 说明 |
|------|------|------|
| 信息 | `[IEMS 信息]` | 正常运行信息 |
| 警告 | `[IEMS 警告]` | 需要注意的问题 |
| 错误 | `[IEMS 错误]` | 操作失败或冲突 |
| 致命错误 | `[IEMS 致命错误]` | 导致游戏崩溃的严重错误 |

### 3.2 能源系统架构

#### 3.2.1 能源单位换算

```
能源单位层级 (从高到低):

  SE (标准能量单位)
   │
   ├─ 1 SE = 100,000,000 GE
   │
   ▼
  GE (通用能量单位)
   │
   ├─ 1 GE = 9,000,000,000,000,000,000 FE (9×10^18)
   │
   ▼
  FE (Forge Energy)
```

#### 3.2.2 精度要求

- **SE 和 GE**: 使用 `BigInteger` 存储
- **显示和存储**: 保留整数，小数向下取整
- **内部计算**: 以 FE 为基准

#### 3.2.3 供电逻辑

```
1. 核心直接供电给所有连接的设备
2. 核心优先使用自身存储的能量
3. 核心能量不足时，向存储器请求能源 (耗电功率×2)
4. 核心能量富余时，向存储器充能
5. 中继器/广播塔无能量缓存，仅标记连接
```

// 存储容量：10^20 GE

```java
// 转换模式:
// 1. SE -> GE (1SE = 10^8 GE)

```

#### 4.2.4 能源中继传输器

```java
// 最大连接距离：500 格
// 连接方式：Shift+ 右键开始连接，右键目标完成连接
// 激光颜色：黄色

```

#### 4.2.5 能源广播塔

```java
// 最大连接距离：50 格
// 自动连接半径：50 格
// 信号源位置：本体向上 3 格
// 扫描区域：朝下的半圆
// 视线检查：≥3 层完整方块遮挡则无法连接
// 无能量缓存
```

### 4.3 广播塔视线检查规则

**核心原则**: 设备任意一个方块没有被完全遮挡即可通电

| 规则 | 说明 |
|------|------|
| **遮挡判定** | 完整方块阻挡≥3 层才算完全遮挡 |
| **不完整方块** | 可以合并计算 (如上下半砖拼起来算 1 层) |
| **透明方块** | 玻璃、树叶等不算遮挡 |
| **发光方块** | 荧石等发光方块**算遮挡** |
| **液体** | 水、岩浆不算遮挡 |
| **手动连接** | 不受视线限制 |

### 4.4 连接规则

**重要**: 中继器和广播塔必须与核心有直接或间接连接才能工作！

```
有效连接:                          无效连接:

核心 ──────→ 广播塔 ✓              广播塔 (未连接) ✗
                                   │
核心 ──────→ 中继器 ──────→ 广播塔 ✓  广播塔 ──→ 中继器 (未连核心) ✗
                                   │
核心 ──────→ 中继器 ──→ 中继器 ──→ 广播塔 ✓  中继器 ──→ 中继器 (未连核心) ✗
```

## 11. 构建说明

### 8.1 核心原则

**设备任意一个方块没有被完全遮挡即可通电**

遮挡判定规则：
- **≥3 层完整方块** = 完全遮挡（无法通电）
- **<3 层完整方块** = 部分遮挡（可以通电）
- **设备方块** = 存储器、转换器、中继器、广播塔**不算遮挡**
- **下半砖/上半砖** = 即使叠放也不算遮挡（因为有缝隙）
- **透明方块** = 玻璃、树叶等不算遮挡
- **液体** = 水、岩浆不算遮挡
- **结构空位** = 标记方块不算遮挡
- **手动连接** = 不受视线限制

### 8.2 算法实现

**文件**: `SightCheckUtil.java`

**算法流程**:
```
1. 从起点到终点发射射线
2. 步长 0.1 格，逐步检查每个位置
3. 检测是否进入新方块
4. 如果是设备方块，跳过
5. 如果是完整方块，记录遮挡
6. 从遮挡方块进入非遮挡方块时，层数 +1
7. 找到 3 层遮挡后立即返回 false
8. 到达终点时未满 3 层返回 true
```

**关键代码**:
```java
// 检测方块边界（进入新方块）
if (!currentBlockPos.equals(lastBlockPos)) {
    if (wasObstructed && !isObstructed) {
        // 从遮挡方块进入非遮挡方块，层数 +1
        obstructionCount++;
    }
    lastBlockPos = currentBlockPos;
}

// 设备方块不算遮挡
if (isDeviceBlock(state)) {
    wasObstructed = false;
    continue;
}
```

**设备方块检测**:
```java
public static boolean isDeviceBlock(BlockState state) {
    String blockName = state.getBlock().toString().toLowerCase();
    
    // IEMS 设备
    if (blockName.contains("energy_storage")) return true;
    if (blockName.contains("energy_converter")) return true;
    if (blockName.contains("energy_relay")) return true;
    if (blockName.contains("energy_broadcast")) return true;
    
    return false;
}
```

### 8.3 示例场景

| 排列方式 | 计算 | 结果 |
|---------|------|------|
| 6 个上半砖叠放 | 有缝隙，不算连续遮挡 | ✓ 可通电 |
| 6 个下半砖叠放 | 有缝隙，不算连续遮挡 | ✓ 可通电 |
| 3 个完整方块 | 1.0×3 = 3.0 层 | ✗ 完全遮挡 |
| 设备 A-B-C- 广播塔 | 设备不算遮挡 | ✓ 可通电 |
| 5 个半砖 | 0.5×5 = 2.5 层 | ✓ 可通电 |
| 2 个完整 +1 个半砖 | 1.0×2 + 0.5 = 2.5 层 | ✓ 可通电 |
| 12 个栅栏 | 0.25×12 = 3.0 层 | ✗ 完全遮挡 |

### 8.4 广播塔扫描逻辑

**文件**: `EnergyBroadcastTowerBlockEntity.java`

**扫描规则**:
1. **信号源位置**: 广播塔本体向上 3 格
2. **扫描区域**: 以信号源为中心，半径 50 格的半球（朝下）
3. **扫描频率**: 每 20 tick（1 秒）扫描一次
4. **自动连接**: 发现有效设备且视线畅通时自动连接

**有效设备类型**:
- EnergyStorageBlockEntity（存储器）
- EnergyConverterBlockEntity（转换器）

**无效设备类型**:
- 其他广播塔
- 中继器（需要手动连接）

## 9. 标记方块系统

### 9.1 概述

**EnergyMarkerBlock** - 一种使用原版结构空位贴图的不可见方块，用于标记机器边缘位置。

**用途**:
- 定义多方块结构的边界
- 标记能量输入/输出点
- 标记连接点位置
- 供其他 MOD 识别和调用

### 9.2 特性

| 特性 | 说明 |
|------|------|
| 渲染 | 结构空位贴图（半透明网格） |
| 碰撞 | 无碰撞箱 |
| 遮挡 | 不参与视线检查 |
| 获取方式 | 仅通过代码放置（不在创造标签页） |
| NBT 绑定 | 可绑定到主机器方块 |

### 9.3 NBT 数据结构

```java
CompoundTag {
    controllerX: int,    // 主机器 X 坐标
    controllerY: int,    // 主机器 Y 坐标
    controllerZ: int,    // 主机器 Z 坐标
    markerType: int,     // 标记类型 (0=边界，1=连接点，2=输入，3=输出)
    markerId: string     // 标记 ID(用于区分同一机器的多个标记)
}
```

### 9.4 标记类型

| ID | 类型 | 说明 |
|----|------|------|
| 0 | 边界标记 | 定义机器边界 |
| 1 | 连接点标记 | 定义连接点位置 |
| 2 | 能量输入标记 | 定义能量输入点 |
| 3 | 能量输出标记 | 定义能量输出点 |

### 9.5 使用示例

**其他 MOD 调用示例**:
```java
// 放置标记方块
BlockPos markerPos = new BlockPos(x, y, z);
level.setBlock(markerPos, IEMSBlocks.ENERGY_MARKER.get().defaultBlockState(), 3);

// 获取标记方块实体
BlockEntity be = level.getBlockEntity(markerPos);
if (be instanceof EnergyMarkerBlockEntity marker) {
    // 绑定到主机器
    marker.setControllerPos(machinePos);
    marker.setMarkerType(2); // 能量输入
    marker.setMarkerId("input_1");
}

// 读取标记信息
BlockPos controllerPos = marker.getControllerPos();
int markerType = marker.getMarkerType();
String markerId = marker.getMarkerId();
```

### 9.6 实现文件

| 文件 | 说明 |
|------|------|
| `EnergyMarkerBlock.java` | 标记方块类 |
| `EnergyMarkerBlockEntity.java` | 标记方块实体（带 NBT） |
| `IEMSBlocks.java` | 注册标记方块和实体 |
