## 渲染方式说明

本模组的方块使用两种渲染方式：

### 1. 1x1x1 方块贴图渲染
使用 Minecraft 原生的方块渲染系统，需要 JSON 模型文件 + PNG 贴图文件。

**适用方块**:
- 标准能量存储器
- 通用能量存储器
- 能量转换器

### 2. MMD 模型渲染
使用 MikuMikuDance (MMD) 的 3D 模型渲染系统，需要 PMX/PMD 模型文件 + 贴图。

**适用方块**:
- 能源中继传输器
- 能源广播塔

## 文件清单

### 一、方块贴图 (textures/block/)

这些贴图用于 1x1x1 方块的表面，Minecraft 会自动应用到对应的 JSON 模型上。

| 文件名 | 对应方块 | 作用 | 尺寸要求 | 设计建议 |
|--------|---------|------|---------|---------|
| `standard_energy_storage.png` | 标准能量存储器 | 方块 6 个面的贴图 | 16x16, 32x32 或 64x64 | 绿色/蓝色科技风格，表示存储 10^5 SE 容量 |
| `general_energy_storage.png` | 通用能量存储器 | 方块 6 个面的贴图 | 16x16, 32x32 或 64x64 | 紫色/金色高级科技风格，表示存储 10^20 GE 容量 |
| `energy_converter.png` | 能量转换器 | 方块 6 个面的贴图 | 16x16, 32x32 或 64x64 | 橙色/黄色，带有转换符号/箭头，表示能源转换功能 |

**注意**: 
- 这 3 个方块使用 Minecraft 原生渲染，不需要额外的模型文件（JSON 模型已提供）
- 贴图会自动应用到方块的 6 个面
- 如需不同面使用不同贴图，需要修改对应的 JSON 模型文件

### 三、MMD 模型文件 (models/entity/)

这些文件用于能源中继传输器和能源广播塔的 3D 模型渲染。

| 文件名 | 对应方块 | 作用 | 格式 | 说明 |
|--------|---------|------|------|------|
| `energy_relay.pmx` | 能源中继传输器 | 3D 模型主体 | PMX 2.0/2.1 | 推荐格式，支持更多特性 |
| `energy_relay.pmd` | 能源中继传输器 | 3D 模型主体 (兼容) | PMD | 旧格式，如无法提供 PMX 时使用 |
| `energy_relay.png` | 能源中继传输器 | 模型表面贴图 | PNG | 配合 PMX/PMD 模型使用，尺寸建议 512x512 或 1024x1024 |
| `energy_broadcast_tower.pmx` | 能源广播塔 | 3D 模型主体 | PMX 2.0/2.1 | 推荐格式，支持更多特性 |
| `energy_broadcast_tower.pmd` | 能源广播塔 | 3D 模型主体 (兼容) | PMD | 旧格式，如无法提供 PMX 时使用 |
| `energy_broadcast_tower.png` | 能源广播塔 | 模型表面贴图 | PNG | 配合 PMX/PMD 模型使用，尺寸建议 512x512 或 1024x1024 |

**MMD 模型技术要求**:

```
坐标系统:
  - Y 轴向上
  - 原点位置：模型底部中心 (x=0, y=0, z=0)
  - 方块占据空间：X(0-1), Y(0-1), Z(0-1)

建议尺寸:
  - 宽度：0.5-0.8 个方块 (8-12.8 像素)
  - 高度：1.5-2.0 个方块 (24-32 像素)
  - 深度：0.5-0.8 个方块 (8-12.8 像素)

材质要求:
  - 材质数量：建议不超过 5 个
  - 顶点数量：建议不超过 5000
  - 骨骼数量：如需要动画，建议不超过 10 个

贴图要求:
  - 格式：PNG (支持透明度)
  - 尺寸：512x512 或 1024x1024 (2 的幂次方)
  - 色彩空间：sRGB
```

**能源中继传输器设计建议**:
- 主题颜色：黄色 (#FFAA00)
- 造型：柱状/塔状，顶部有信号发射器
- 细节：电路板纹理、发光效果
- 最大连接距离：500 格 (可在模型上体现长距离传输的视觉元素)

**能源广播塔设计建议**:
- 主题颜色：蓝色 (#0088FF)
- 造型：塔状，顶部有球形/碟形信号发射器
- 细节：无线信号波纹、科技感装饰
- 自动连接半径：50 格 (可在模型上体现短距离无线的视觉元素)

### 四、GUI 贴图 (textures/gui/)

这些贴图用于游戏内 GUI 界面。

| 文件名 | 用途 | 作用 | 尺寸要求 | 设计说明 |
|--------|------|------|---------|---------|
| `emd_gui.png` | EMD 能源监控面板 | GUI 背景底图 | 256x256 | 科幻风格，包含边框、插槽区域、背景网格等 |
| `circuit_pattern.png` | 电路板花纹 | EMD 界面装饰 | 64x64 (可平铺) | 橙色电路板触点花纹，用于背景装饰，支持平铺 |

**EMD GUI 布局说明**:

```
┌─────────────────────────────────────┐
│  ┌─────────────┐                    │
│  │ 当前电量    │   [电路板花纹区域]  │
│  │ xxxSE       │   橙色脉冲动画效果  │
│  │             │                    │
│  │ xx:xx 充满  │                    │
│  │             │                    │
│  │ 耗电  发电  │                    │
│  └─────────────┘                    │
│                                     │
│  [玩家物品栏区域]                    │
└─────────────────────────────────────┘
```

**emd_gui.png 设计要点**:
- 整体尺寸：256x256 像素
- 背景颜色：深色科技风格 (深灰/深蓝)
- 信息框位置：左上角 (约 120x80 像素区域)
- 物品栏位置：底部 (与 Minecraft 标准物品栏对齐)
- 装饰元素：科技线条、网格、发光效果

**circuit_pattern.png 设计要点**:
- 尺寸：64x64 像素 (可无缝平铺)
- 主色调：橙色 (#FF8800)
- 图案：电路板走线、触点、芯片轮廓
- 透明度：部分区域透明，用于叠加效果

### 五、实体贴图 (textures/entity/)

这些贴图用于游戏中动态实体（如激光连接）的渲染。

| 文件名 | 用途 | 作用 | 尺寸要求 | 设计说明 |
|--------|------|------|---------|---------|
| `energy_connection.png` | 能量连接激光 | 正常状态激光贴图 | 64x64 | 黄色激光束，支持透明度，用于设备连接时的视觉效果 |
| `energy_connection_depleted.png` | 耗尽状态激光 | 电网耗尽时激光贴图 | 64x64 | 红色激光束，表示电网已耗尽，警告效果 |

**激光贴图技术要求**:

```
尺寸：64x64 像素 (或 128x128 以获得更高质量)
格式：PNG (必须支持 Alpha 透明度通道)
色彩模式：RGBA
内容：
  - 能量连接.png: 黄色 (#FFAA00) 激光束，从中心向两侧渐变透明
  - 能量连接_depleted.png: 红色 (#FF0000) 激光束，闪烁效果
```

**使用场景**:
- 当能源中继传输器/广播塔连接到其他设备时，会在两者之间渲染激光束
- 正常状态：黄色激光
- 电网耗尽状态：红色激光（警告玩家）

## 文件命名规范

1. **全部使用小写字母**
2. **单词间使用下划线分隔**
3. **不要使用空格或特殊字符**
4. **保持与代码中的注册名称一致**

**正确示例**:
- `standard_energy_storage.png` ✓
- `energy_relay.pmx` ✓

**错误示例**:
- `StandardEnergyStorage.png` ✗ (大写)
- `energy-relay.png` ✗ (连字符)
- `energy relay.png` ✗ (空格)

## 提交文件时的注意事项

1. **贴图文件**:
   - 确保 PNG 格式
   - 检查透明度通道是否正确
   - 确认尺寸为 2 的幂次方 (16, 32, 64, 128, 256, 512, 1024)

2. **MMD 模型文件**:
   - PMX 格式优先
   - 检查模型坐标原点是否在底部中心
   - 确认贴图路径正确
   - 测试模型在游戏中的渲染效果

3. **文件放置位置**:
   - 严格按照目录结构放置
   - 文件名必须完全匹配（包括大小写）
